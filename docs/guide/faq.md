# 常见问题（FAQ）

该部分主要提供 MAK（雪松小助手）相关的概念性解答。

> ⚠️ **遇到运行报错或功能异常？** 请先查阅 [问题排查手册](./trouble-shooting.md)，那里有更详细的排查步骤。

---

## MXU（界面）

::: details 什么是 MXU？

**MXU**（MaaFramework Next UI），是一个基于 [MaaFramework PI V2](https://github.com/MaaXYZ/MaaFramework/blob/main/docs/zh_cn/3.3-ProjectInterfaceV2%E5%8D%8F%E8%AE%AE.md) 协议的通用 GUI 客户端。它为本项目提供了开箱即用的图形界面。

:::

::: details 为什么选择 MXU？

开发者结合自己参与 [MaaNTE](https://github.com/1bananachicken/MaaNTE) 开发的经验，在参考了 [M9A](https://github.com/MAA1999/M9A)、[MaaEnd](https://github.com/MaaEnd/MaaEnd) 等项目后，决定采用 MXU。

对我们来说，MXU 有配置简单、界面交互方便、日志管理优秀等优点。虽然仍有些许不足之处，但它是目前主流 MaaFramework GUI 中最为合适的。

:::

### 配置/标签页

::: details 什么是配置/标签页？

MXU 将 MaaFramework 配置以标签页的形式展示，以便用户选择合适的配置用于不同用途。每个标签页相当于一套独立的配置，你可以为不同的场景创建不同的配置。

:::

::: details 配置包括的范围有哪些？

MXU 描述为"各配置的任务和设备设置互相独立"。实际上：

- **任务独立**：任务列表、任务详细配置、任务启用状态、任务展开状态等各自独立
- **定时功能独立**：每个配置的定时执行设置互不影响
- **连接设置独立**：控制器、连接窗口、实时截图显示状态等各自独立
- **全局设置共享**：设置中的内容各配置共享

:::

::: details 关于预制配置

我们在未来将会将部分常用配置整合为几份预设，在首次启动软件时会自动应用。如果不需要的话，也可以通过关闭标签页的方式来删除该配置。

如果你误删了预设配置，想要将其重新添加回来，只需要在新建配置时选择对应预设即可。

:::

::: details 关于定时执行配置

在配置的右下角，可以通过定时功能让某个配置定时执行。该功能各配置相互独立。

:::

### 任务

::: details 什么是任务？

MAK 中各个自动化功能作为独立的"任务"，你可以根据需要自由启用或禁用。每个任务对应游戏中的一个具体操作或玩法。

你可以在任务列表中勾选需要执行的任务，MAK 会按列表顺序依次执行。

:::

::: details 任务的执行顺序是怎样的？

任务是**按任务列表从上到下依次执行**的，会跳过未勾选的任务。你可以通过拖拽任务标题左侧的六个点来调整顺序。

:::

::: details 为什么不支持同时执行多个任务？

MAK 通过图像识别与模拟键鼠操作来实现自动化，多个任务同时执行会导致操作互相干扰。因此 MAK 设计为顺序执行。

:::

::: details 什么是控制器？各控制器有什么区别？

控制器是 MaaFramework 中用于与游戏窗口交互的组件。MAK 提供了一种控制器——`ADB`

你可以在右上角的连接设置中选择合适的控制器。

:::

::: details 如何知道任务是否正常运行？

任务执行过程中，你可以在 MAK 的日志面板中查看实时运行日志。如果任务出现异常或失败，日志中会有相应的错误提示。

:::

## MFAA（界面）

::: details 什么是 MFAA？

**MFAA**（MFAAvalonia）是基于 [Avalonia UI](https://github.com/AvaloniaUI/Avalonia) 构建的 [MaaFramework](https://github.com/MaaXYZ/MaaFramework) 通用 GUI。它为本项目提供了另一种开箱即用的图形界面。

:::

::: details 为什么提供 MFAA？

MFAA 和 MXU 支持相同的 MAK 任务，但界面布局和操作方式有所不同。MFAA 将任务列表、任务设置、实时视图和日志分区展示，方便用户在同一页面查看任务配置与运行状态。

提供 MFAA 是为了让用户可以按照自己的使用习惯选择界面。如果当前界面无法正常运行，也可以尝试改用另一种界面。

:::

### 配置/标签页

::: details 什么是配置/标签页？

MFAA 和 MXU 一样，也会在窗口顶部以标签页展示配置。每个标签页相当于一套配置方案，你可以为不同的使用场景创建不同配置。

在每个配置中，你都可以选择资源类型、控制器类型和当前控制器，并管理该配置的任务列表。

:::

::: details 如何切换或新建配置？

你可以直接点击窗口顶部的配置标签页进行切换，点击标签页右侧的 **+** 可以新建配置。

也可以进入左下角的“设置”→“切换配置”：

- 在“当前配置”中查看或切换已有的配置方案
- 填写“新增配置名称”后，点击“添加”，基于当前配置创建新的配置

:::

::: details 关于定时执行配置

进入“设置”→“定时执行”，可以为配置设置定时执行计划，并在“自定义配置选择”中指定需要执行的配置。

开启“强制定时启动”后，如果定时触发时已有任务正在运行，MFAA 会先停止当前任务，再在停止完成后开始新的任务。

:::

::: details MFAA 主界面有哪些区域？

MFAA 左侧提供以下主要页面入口：

- **主页**：管理当前配置的资源、控制器和任务
- **总控台**：集中查看和管理运行内容
- **截图**：使用截图相关功能
- **设置**：修改配置、运行和界面等选项

“主页”中的配置标签页主要包括以下区域：

- **资源与控制器**：位于页面顶部，用于选择资源类型、控制器类型和当前控制器
- **任务列表**：位于左侧，用于添加、启用和调整任务顺序
- **任务设置**：位于中间上方，用于修改当前任务的选项
- **任务说明**：位于中间下方，用于查看当前任务的说明
- **实时视图**：位于右侧上方，用于查看设备截图
- **日志**：位于右侧下方，用于查看连接、任务执行和错误信息

:::

::: details 设置中可以调整哪些内容？

点击左下角的“设置”可以看到以下分类：

- **切换配置**：切换已有配置，或基于当前配置新建配置
- **定时执行**：设置配置的定时执行计划
- **性能设置**：调整性能相关选项
- **运行设置**：调整任务运行相关选项
- **连接设置**：调整设备连接相关选项
- **启动设置**：调整软件启动时的相关选项
- **界面设置**：调整界面显示相关选项
- **外部通知**：配置外部通知
- **热键设置**：配置快捷键
- **更新设置**：调整更新相关选项
- **关于我们**：查看软件信息

:::

::: details 什么是资源类型？

资源类型用于选择游戏的渠道版本。MAK 当前提供以下三种资源类型：

- **官服**：适用于从官网下载的游戏
- **B 服**：在通用资源的基础上加载 Bilibili 渠道资源
- **TapTap**：在通用资源的基础上加载 TapTap 渠道资源

请根据你安装的游戏版本，在配置页顶部选择对应的资源类型。

:::

### 任务

::: details 什么是任务？

MAK 中各个自动化功能作为独立的“任务”，你可以根据需要自由启用或禁用。每个任务对应游戏中的一个具体操作或玩法。

你可以在左侧任务列表中添加和勾选需要执行的任务。选中任务后，中间区域会显示该任务的设置和说明。

:::

::: details 任务的执行顺序是怎样的？

任务是**按任务列表从上到下依次执行**的，会跳过未勾选的任务。你可以直接在任务列表中拖拽任务来调整执行顺序。

:::

::: details 为什么不支持同时执行多个任务？

MAK 通过图像识别与模拟键鼠操作来实现自动化，多个任务同时执行会导致操作互相干扰。因此 MAK 设计为顺序执行。

:::

::: details 什么是控制器？如何选择控制器？

控制器是 MaaFramework 中用于与游戏窗口交互的组件。MAK 当前提供一种控制器：`ADB`。

在配置页顶部的“控制器类型”中选择 `ADB 控制器`，再在“当前控制器”中选择需要连接的模拟器或设备。

:::

::: details 如何知道任务是否正常运行？

任务执行过程中，可以在右侧的“实时视图”中查看设备画面，并在“日志”中查看连接状态和实时运行信息。如果任务出现异常或失败，日志或界面通知中会显示相应的错误提示。

:::

## 使用相关

::: details 模拟器分辨率应该设置多少？

MAK **仅支持 16:9 的分辨率**。推荐以下两种：

- `1280×720`（720P）
- `1920×1080`（1080P）

    ::: warning 注意
    是在**模拟器的系统设置**中修改分辨率，不是在游戏内设置！
    :::

:::

::: details MAK 操作的时候，我可以动电脑吗？

可以，但是不可以动模拟器。MAK 运行时会自动点击屏幕和识别画面。如果你动了模拟器，MAK 就可能出错。

建议：在 MAK 运行的时候，你可以去用别的软件，或者休息一会，等它完成再说。

:::

::: details MAK 支持哪些游戏版本？

目前支持《雪松》**官服**（从官网下载的版本）。B 服和其他渠道服也许会有一点适配，理论上也能使用。

:::

::: details 功能太少了，什么时候加新功能？

本项目目前处于早期开发阶段，开发者在用业余时间维护。新功能会陆续添加，敬请关注 [GitHub Release](https://github.com/Hollow-YK/MaaAssistantKedrgame/releases) 页面获取更新。

:::

::: details 会封号吗？

MAK 只是模拟你的手指点击屏幕，不会修改游戏任何数据，本质上和你自己手动操作一样。

但根据游戏用户协议，使用第三方辅助工具存在一定风险。**请自行评估风险后使用**，本项目不对任何后果负责。

:::

::: details 我是 Mac 电脑，能用吗？

目前 MAK 只支持 Windows 系统，Mac 暂不支持。

:::
