## 什么是 ADB？

ADB（Android Debug Bridge，Android 调试桥）是 Google 官方提供的一个多功能命令行工具，用于与 Android 设备（包括模拟器和真机）进行通信。它包含了三个组件：

- **客户端**：运行在电脑上，用于发送 ADB 命令
- **守护进程（adbd）**：运行在 Android 设备上，负责执行命令
- **服务端**：运行在电脑后台，管理客户端与守护进程之间的通信

通过 ADB，你可以执行安装应用、运行 shell 命令、传输文件、截屏等操作。

### MAK 的 ADB 工作原理

MAK 基于 **MaaFramework** 实现自动化操作，而 MaaFramework 底层通过 ADB 与 Android 设备通信。整个工作流程如下：

``` mermaid
flowchart LR
    A[MAK<br/>主程序] -->|1. 下发任务| B[MaaFramework]
    B -->|2. ADB 指令| C[ADB 客户端]
    C -->|3. USB / 网络| D[Android 设备<br/>模拟器 / 真机]
    D -->|4. 截屏| E[屏幕图像]
    E -->|5. 返回图像| B
    B -->|6. 图像识别| F[识别结果]
    F -->|7. 计算点击位置| G[生成点击指令]
    G -->|8. ADB 模拟输入| D
```

具体来说：

1. **截图**：MaaFramework 通过 ADB `screencap` 命令获取设备当前屏幕图像
2. **识别**：MaaFramework 使用内置的图像识别算法（如模板匹配、特征检测等），在截图中找到目标按钮或界面的位置
3. **点击**：通过 ADB `input tap` 命令模拟触摸点击
4. **循环**：不断重复上述步骤，形成一个完整的自动化流程

MAK 负责定义任务流程和识别目标（如按钮截图），MaaFramework 负责执行截图、识别、点击等底层操作。

因此，**只要 MaaFramework 能通过 ADB 连接的设备，MAK 就能操控**——无论是模拟器还是真机。

## 使用 ADB 连接模拟器

MAK 会自动检测并列出当前运行中的模拟器设备。你只需在设备列表中选择目标模拟器即可连接，**无需手动输入 ADB 地址**。

### 查找模拟器 ADB 端口

如果同时运行了多个模拟器，了解各模拟器的默认 ADB 端口可以帮助你在设备列表中快速识别目标设备：

::: note 注意
下面的数据来源于互联网，可能存在不准确之处，请以实际情况为准。
:::

| 模拟器 | 默认 ADB 端口 | 多开端口规律 | 备注 |
|-------|-------------|------------|------|
| MuMu 模拟器 12 | `16384` | **+32**（实例 0=16384, 1=16416, 2=16448, …） | 可在 MuMu 多开器右上角 ADB 图标查看 |
| MuMu 模拟器 6（旧版） | `7555` | — | 较老版本的 MuMu 使用此端口 |
| 雷电模拟器 | `5555` | **+2**（实例 0=5555, 1=5557, 2=5559, …） | 设备名显示为 `emulator-5554` 等 |
| 夜神模拟器 | `62001` | `59865`（多开） | 可在夜神多开器查看各实例端口 |
| 逍遥模拟器 | `21503` | — | 可在模拟器设置中查看 |
| BlueStacks 蓝叠 5 | `5555` | 多个预设端口：`5555/5556/5565/5575/5585/5595` | **Hyper-V 版端口随机**，需查看配置文件 |
| 腾讯手游助手 | `5555` | — | 端口固定 |

::: tip 如何确认端口号？
打开模拟器的「**设置**」→「**其他设置**」或「**ADB 调试**」，一般会显示 ADB 端口号。也可以在命令行运行 `adb devices` 查看当前已连接的设备及其端口。

知道端口号后，在 MAK 的设备列表中查找对应端口的设备即可。
:::

### 连接步骤

1. **启动模拟器**：确保你的模拟器已经正常打开
2. **打开 MAK**：进入主界面
3. **选择设备**：在设备列表中，找到你的模拟器并点击选中
4. **开始使用**：连接成功后即可添加任务并开始运行

### 常见问题

- **模拟器未显示**：检查模拟器是否已开启 ADB 调试功能，或尝试重新扫描
- **连接被拒绝**：尝试重启模拟器，或在模拟器设置中重新开启 ADB
- **端口冲突**：如果运行了多个模拟器，端口可能会冲突，关闭多余的模拟器后重试
- **设备列表为空**：确认模拟器已正常启动，且 ADB 调试已开启

## 使用 ADB 连接真机

如果你希望直接在手机上使用 MAK，也可以通过 ADB 连接你的 Android 真机。

::: caution 危险功能
通过真机使用 MAK 属于极度危险的行为，你应确保知道自己在干什么！

通过真机使用 MAK 造成的任何后果均应由你自己承担，MAK 项目及开发者不为此承担任何责任。

请在通过真机使用 MAK 前，了解可能存在的全部风险，并确保你知道如何操作以及发生意外后如何挽救损失！
:::

### 前置条件

1. **开启开发者选项**：在手机「设置」→「关于手机」→ 连续点击「版本号」7 次（不同厂商可能略有不同）
2. **开启 USB 调试**：「设置」→「开发者选项」→ 打开「USB 调试」
3. **使用数据线**将手机连接到电脑

### 有线连接步骤

1. 用数据线将手机连接到电脑
2. 手机上第一次连接会弹出「**允许 USB 调试吗？**」的提示，勾选「**始终允许**」，然后点击「**确定**」
3. 打开 MAK
4. 在设备选择界面，找到你的手机设备并点击选中
5. 如果未自动出现，可以尝试重新插拔数据线，或重启 MAK 重新扫描

### 无线连接步骤（免数据线）

如果你觉得数据线太麻烦，手机和电脑在**同一网络**下时，可以使用无线调试。

具体步骤略。

### 常见问题

- **设备未显示**：检查 USB 调试是否已开启，或尝试重新插拔数据线
- **连接失败**：部分手机需要安装手机厂商的 USB 驱动程序，可以在手机官网下载
- **驱动问题**：如果电脑无法识别手机，可以尝试安装 [Android USB 驱动](https://developer.android.com/studio/run/win-usb) 或用驱动管理软件更新驱动
- **无线连接不稳定**：确保手机和电脑处于同一网络，且信号良好；有线连接会更稳定
